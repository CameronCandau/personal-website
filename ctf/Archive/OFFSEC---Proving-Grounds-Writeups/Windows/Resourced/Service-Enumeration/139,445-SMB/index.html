Password Audit sounds promising and contains two directories, ‘Active Directory’ and ‘Registry’.

‘Active Directory’ has ntds.dit, which is the database file which holds all authentication information for an active directory domain. This is a great finding for us.

We also have the SECURITY and SYSTEM hives under the Registry Directory. I’ll download these as well. When re-attempting to download SYSTEM on a subsequent solve, I encountered timeout issues, since it’s a much larer file than SECURITY. I fixed this by setting a higher timeout like -t 600:

To extract useful secrets from the database, we can use ntds.dit, SYSTEM, and SECURITY together back on our attacking system.

impacket-secretsdump -ntds ntds.dit -system SYSTEM LOCAL
